OFBiz Security

OFBiz Security Model: Permissions and related. Be sure to read the children pages in the left part of screen.
Keeping OFBiz secure. To keep your OFBiz instance secure from exploits. Also read the children pages, notably How to secure your deployment.
In the main online README file, best practices and security advices for OFBiz users in production.

All system privileges, including access to potentially vulnerable operations, are granted to administrators. Even if we assume that administrators don't attack their own websites, it's essential to exercise extra care when granting administrator privileges. Therefore, if a security breach occurs on the administration page (webtools), it's generally not perceived as a problem. The administrator holds the power. Unless an ordinary user manages to overstep their bounds and act beyond their authority. So in the webtools page we only accept vulnerabilities when using a not administrator credential.

At the UI level the OFBiz logs are protected and should not be vulnerable to exploits. We though warn OFBiz users it's important that out of OFBiz UI level logs files remain restricted to their trusted users. Also we recommend to use the verbose level on production only when it's absolutely necessary.

Security Vulnerabilities

We strongly encourage OfBiz users to report security problems affecting OFBiz to the private security mailing lists (either preferably security@ofbiz.apache.org or else security@apache.org), before publicly disclosing them once fixed. Please don't pack several vulnerabilities in the same report, send them one by one, thanks in advance.

Please see the ASF Security Team webpage for further information about reporting a security vulnerability as well as their contact information.
